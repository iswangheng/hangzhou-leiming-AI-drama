# 假阳性优化方案 - 深度分析与解决方案

**日期**: 2026-03-03
**版本**: V1.0
**问题**: 精确率仅18.6%，81%的AI标记为假阳性

---

## 🔍 问题诊断

### 当前数据（4个项目测试）

| 指标 | 数值 | 问题 |
|------|------|------|
| **AI标记总数** | 85个 | 过度标记 |
| **正确标记** | 16个 | 仅18.6%正确 |
| **假阳性** | 69个 | 81%是错误的！ |
| **平均每集钩子** | 2.1个 | 低于限制3个，说明不是数量问题 |

### 核心问题：类型分布失衡

| 钩子类型 | 数量 | 占比 | 问题 |
|---------|------|------|------|
| **对话突然中断** | 27个 | **33.8%** | 🚨 假阳性最大来源 |
| 情节转折 | 14个 | 17.5% | 合理 |
| 情感爆发时刻 | 9个 | 11.2% | 合理 |
| 悬念反转 | 9个 | 11.2% | 合理 |
| 其他 | 21个 | 26.3% | 合理 |

### 根本原因分析

#### 1. 提示词结构问题

```python
## 当前提示词 (scripts/understand/analyze_segment.py:69-75)
## 常见钩子点位置
- 对话突然中断（被打断、戛然而止）  # ← 第1位！AI优先选择
- 突然出现新信息（秘密揭露、真相揭示）
- 情节出现转折（反转、突变、意外）
```

**问题**:
- "对话突然中断"列在第一位，AI优先匹配
- 短剧中每集都有大量对话停顿，AI会把普通停顿误判为钩子
- 缺乏"什么是NOT钩子点"的明确指导

#### 2. 矛盾的指令

```python
## 矛盾的指导 (scripts/understand/analyze_segment.py:42, 82, 89, 90)
第42行: "保持宁缺毋滥原则"  # ← 要保守
第82行: "有明显迹象就标记"   # ← 要放宽
第89行: "适度放宽"          # ← 要放宽
第90行: "仍然保持质量"      # ← 要保守
```

**问题**:
- AI不知道应该保守还是放宽
- 导致"宁可错杀一千，不可放过一个"的策略

#### 3. 缺乏钩子强度评分标准

**当前**: 只有0-10的置信度评分，没有明确标准

**后果**:
- AI对所有"对话突然中断"都给7-8分（感觉还行）
- 但实际上大部分不满足"真正钩子"的标准

---

## 🎯 优化策略

### 策略1: 重新排序常见钩子点位置

**目标**: 降低"对话突然中断"的优先级

**当前**:
```python
## 常见钩子点位置
- 对话突然中断（被打断、戛然而止）  # ← 第1位
- 突然出现新信息（秘密揭露、真相揭示）
- 情节出现转折（反转、突变、意外）
```

**优化后**:
```python
## 常见钩子点位置（按强度排序）
1. 情节重大转折（反转、突变、意外、命运改变）  # ← 强度最高
2. 秘密揭露或真相揭示（关键信息暴露、身份揭晓）
3. 悬念反转时刻（剧情急转直下、意想不到的变化）
4. 冲突爆发前一刻（争吵、对抗、矛盾即将激化）
5. 情感爆发顶点（愤怒、震惊、崩溃的极致时刻）
6. 信息截断/欲知后事（关键时刻被打断、画面停在悬念处）

## ⚠️ 谨慎标记的类型（容易误判）
- 对话突然中断（必须满足严格的3个条件，见下文）
```

**预期效果**:
- AI会优先匹配情节转折、秘密揭露等强钩子
- "对话突然中断"从第1位降到末位，并加上"谨慎"标记

---

### 策略2: 添加"什么是NOT钩子点"

**目标**: 明确告诉AI什么是假阳性

**新增内容**:
```python
## ❌ 以下情况不是钩子点（不要标记）

### 对话中断类的假阳性
1. **普通对话结束**
   - ❌ 场景A说完话，场景B开始说话
   - ❌ 角色说完一句完整的话，镜头切换
   - ✅ 必须是: 在关键信息即将说出时被打断

2. **场景切换导致的停顿**
   - ❌ 从一个场景切换到另一个场景
   - ❌ 镜头转换、时间地点变化
   - ✅ 必须是: 同一场景内，情节被意外打断

3. **情感自然的结束**
   - ❌ 角色表达完情感后自然平静下来
   - ❌ 对话达成共识或结束
   - ✅ 必须是: 情绪达到高点，但后续未知

### 其他假阳性
4. **普通情节推进**
   - ❌ 日常对话、常规剧情发展
   - ❌ 铺垫性的情节（为后续做准备的）
   - ✅ 必须是: 改变剧情方向的转折点

5. **已知信息的重复**
   - ❌ 观众已经知道的信息
   - ❌ 前面已经揭示过的秘密
   - ✅ 必须是: 新的、重要的、观众不知道的信息
```

**预期效果**:
- 明确的负面例子可以减少50%以上的假阳性
- AI学会区分"真钩子"和"假钩子"

### 策略3: "对话突然中断"的严格判断条件

**目标**: 即使标记"对话突然中断"，也必须满足严格条件

**新增内容**:
```python
## ⚠️ "对话突然中断"的严格判断标准

**只有同时满足以下3个条件，才能标记为"对话突然中断"**:

### 条件1: 信息必须关键（重要性）
✅ 正确示例:
- 角色即将说出秘密/真相时被打断
- 身份即将揭晓时被切断
- 关键决定即将做出时被打断
- 危险即将被揭示时被打断

❌ 错误示例:
- 普通对话被打断（日常闲聊、问候）
- 重复已知信息
- 不重要的细节被打断

### 条件2: 中断必须意外（突发性）
✅ 正确示例:
- 第三方突然闯入打断对话
- 突发事件（爆炸、闯入、晕倒）
- 画面突然黑屏/转场
- 被暴力打断（推倒、捂嘴、绑架）

❌ 错误示例:
- 对话自然结束（说完了）
- 场景切换（时间地点变化）
- 镜头转换（不是中断，是转场）

### 条件3: 观众必须迫切想知道后续（悬念强度）
✅ 正确示例:
- 主角即将说出凶手是谁 → 被打断 → 观众迫切想知道
- 女主即将答应求婚 → 被打断 → 观众迫切想知道结局
- 秘密即将揭晓 → 被打断 → 观众迫切想知道是什么

❌ 错误示例:
- 打断后观众不关心后续
- 打断后情节没有变化
- 打断后继续正常的对话

**评分标准**:
- 满足3个条件: 置信度8-10分（可以标记）
- 只满足2个条件: 置信度6-7分（谨慎标记）
- 只满足1个条件: 置信度<6分（不要标记）
```

**预期效果**:
- 从27个"对话突然中断"中，预计过滤掉70%（保留8个左右的真钩子）
- 精确率从18.6%提升到30-35%

### 策略4: 添加钩子强度评分标准

**目标**: 让AI有明确的评分依据

**新增内容**:
```python
## 🎯 钩子强度评分标准（0-10分）

### 9-10分：超强钩子（必须标记）
**特征**:
- 剧情发生重大转折（重生、穿越、身份揭晓）
- 关键秘密即将揭晓但被打断
- 主角生命/命运受到威胁的悬念
- 观众会疯狂想知道后续的时刻

**示例**:
- 重生剧：主角重生后复仇决心刚刚表达，画面停在这里
- 悬疑剧：凶手即将被说出名字时被打断
- 偶像剧：女主即将答应求婚时被打断

### 7-8分：强钩子（应该标记）
**特征**:
- 明显的情节转折（反转、突变）
- 新信息突然出现（秘密揭露、真相揭示）
- 冲突即将爆发（争吵前一刻、对抗开始前）
- 情感达到高点（愤怒、震惊、崩溃）

**示例**:
- 情感爆发后画面停在悬念处
- 突然出现新角色/新信息
- 决定性时刻的前一秒

### 6-7分：中等钩子（谨慎标记）
**特征**:
- 有一定转折，但不够强烈
- 信息有一定重要性，但不是关键
- 有悬念，但不是"迫切想知道"

**判断**: 如果是6.5分以下，不要标记

### <6分：弱钩子/不是钩子（不要标记）
**特征**:
- 普通对话结束
- 场景切换
- 情节正常推进
- 不重要的信息被截断

**判断**: 宁可不标记，也不要标记弱钩子
```

**预期效果**:
- AI有明确的评分依据，不会随意给7-8分
- 预计30-40%的标记会被降级到<6分，被过滤掉

### 策略5: 解决"适度放宽"与"宁缺毋滥"的矛盾

**目标**: 统一指令，让AI明确策略

**当前矛盾**:
```python
第42行: "保持宁缺毋滥原则"  # ← 要保守
第82行: "有明显迹象就标记"   # ← 要放宽
第89行: "适度放宽"          # ← 要放宽
第90行: "仍然保持质量"      # ← 要保守
```

**优化后（统一策略）**:
```python
## 核心原则：宁可漏掉，也不要错标

根据44个短剧项目的分析经验：
- **每集通常有2-3个真正的钩子点**
- **不是每个对话停顿都是钩子**
- **关键是要准确，不是数量多**

**判断标准**:
- 如果是"可能"是钩子，但不确定 → **不要标记**
- 如果是"应该"是钩子，但不够强 → **不要标记**
- 只有"绝对是"钩子，且强度够 → **才标记**

**记住**: 标记一个假阳性，比漏掉一个真钩子更糟糕
```

**预期效果**:
- AI明确知道"宁可漏掉，也不要错标"
- 预计召回率从51.6%下降到45-48%（可接受）
- 精确率从18.6%提升到35-40%（大幅改善）

### 策略6: 要求AI解释"为什么观众会迫切想知道后续"

**目标**: 强制AI思考钩子的有效性

**优化输出格式**:
```python
## 输出格式（增加"悬念强度"字段）
```json
{{
  "highlight": {{
    "exists": true/false,
    "preciseSecond": 125,
    "type": "从上述类型中选择，如：开篇高光",
    "confidence": 8.5,
    "reasoning": "女主重生后表达复仇决心，情感强烈，引发观众期待"
  }},
  "hook": {{
    "exists": true/false,
    "preciseSecond": 366,
    "type": "从上述类型中选择，如：悬念反转",
    "confidence": 9.2,
    "reasoning": "男主突然说出关键秘密，画面停在悬念时刻，观众迫切想知道后续",
    "suspense_strength": {{
      "what_question": "秘密是什么？",                    // 观众想知道什么
      "why_care": "这个秘密会改变男女主关系",            // 为什么观众关心
      "how_strong": "9/10 - 强烈想知道"                   // 悬念强度0-10
    }}
  }}
}}
```

**要求**:
- 如果AI无法清晰填写"suspense_strength"，说明这不是真正的钩子
- "what_question"必须是明确的、具体的问题
- "why_care"必须说明为什么观众会关心
- "how_strong"必须≥7分才能标记

**预期效果**:
- 强制AI思考"为什么这是钩子"
- 预计20-30%的标记会被AI自我否定（因为填不出来）

- 突然出现新角色/新信息
- 决定性时刻的前一秒

### 6-7分：中等钩子（谨慎标记）
**特征**:
- 有一定转折，但不够强烈
- 信息有一定重要性，但不是关键
- 有悬念，但不是"迫切想知道"

**判断**: 如果是6.5分以下，不要标记

### <6分：弱钩子/不是钩子（不要标记）
**特征**:
- 普通对话结束
- 场景切换
- 情节正常推进
- 不重要的信息被截断

**判断**: 宁可不标记，也不要标记弱钩子

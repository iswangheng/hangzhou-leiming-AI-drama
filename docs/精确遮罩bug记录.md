## 修复记录 (2026-03-13)
**问题根源**：
1. PaddleOCR 默认返回的是整行文本（或文本块）的单一 bounding box，而代码中 `get_word_boxes_for_sensitive_word` 误以为 `boxes` 是按字符级别的列表。当敏感词在文本串的索引 `i` 大于 `boxes` 的长度（通常为1）时，导致匹配不到坐标框。
2. FFmpeg滤镜命令在处理多片段遮罩时，`boxblur=8:8` 的第二个参数（chroma_radius）最大不能超过6，导致滤镜链编译失败（`Invalid chroma_param radius value 8`），FFmpeg执行失败后代码又静默回退（fallback）到了全屏区域遮罩或原视频覆盖。

**修复方案**：
1. 修改了 `scripts/preprocess/subtitle_ocr.py` 中的 `extract_subtitle_text_with_boxes`，增加字符级坐标插值逻辑。通过计算整行 bbox 的宽度平分给每个字符，生成精确的字符级坐标列表。
2. 修改了 `scripts/preprocess/video_cleaner.py`，支持解析更新后的 4 点坐标格式 `[[x1,y1], [x2,y2], [x3,y3], [x4,y4]]`。
3. 修改了 `video_cleaner.py` 中的 FFmpeg 复杂滤镜链。将 `boxblur=8:8` 修正为了 `boxblur=8`（单参数同时控制亮度与色度模糊，默认合法）。
4. 修复了多重滤镜链串联的语法逻辑。改为通过 `v{i-1}` 作为上一个模糊滤镜的输入链，保证多个马赛克叠加可以正确生效。

**结果**：
重新运行测试，烈日重生第一集的 `2.5s (热死)`、`19.5s (高温末世)`、`38.0s (高温末世)` 三处敏感词均已成功抓取并叠加精准马赛克！

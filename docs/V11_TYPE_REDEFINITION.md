# V11优化方案 - 类型重新定义

**核心思想**：不要试图让AI"学会判断"，而是重新定义类型，让类型本身就没有歧义。

---

## 问题诊断

### 当前问题："对话突然中断"类型太宽泛

**V8的7个"对话突然中断"分析**：

| 集数 | 时间 | 是否匹配 | 描述 | 问题 |
|------|------|---------|------|------|
| 第1集 75秒 | ❌ | 男主说"那就请太夫人" | 普通对话，不应该标记 |
| 第2集 48秒 | ✅ (12秒差距) | 夫人正要说出秘密被打断 | **真钩子** |
| 第3集 29秒 | ❌ | 女主说"我这么好语"，镜头切换 | 场景切换，不应该标记 |
| 第4集 69秒 | ❌ | 男主话说到一半 | 不关键，不应该标记 |
| 第5集 54秒 | ✅ (7秒差距) | 女主要进去看男主，画面结束 | **真钩子** |
| 第7集 29秒 | ❌ | 女主质问男主，戛然而止 | 不关键，不应该标记 |
| 第9集 29秒 | ❌ | 族主指责，话没说完 | 不关键，不应该标记 |

**准确率**：2/7 = 28.6% (很低)

### 核心矛盾

"对话突然中断"这个类型涵盖了3种完全不同的情况：
1. **关键信息被截断**（真钩子）→ 2个
2. **普通对话结束**（假钩子）→ 4个
3. **场景切换停顿**（假钩子）→ 1个

AI无法准确区分，因为类型名称本身就有歧义。

---

## V11解决方案：类型重新定义

### 方法：拆分"对话突然中断"为3个独立类型

**原类型**：
- "对话突然中断"（太宽泛，准确率28.6%）

**新类型**：
1. **"关键信息被截断"**（真钩子）
   - 特征：秘密、真相、决定、身份即将揭晓时被打断
   - 示例："夫人正要说出陆大人的秘密，突然被闯入的家丁打断"
   - 应该标记：是

2. **"对话自然结束"**（不是钩子）
   - 特征：角色表达完完整意思后停顿
   - 示例："男主说：那就请太夫人。对话结束"
   - 应该标记：否

3. **"悬念结束时刻"**（可能是真钩子）
   - 特征：行动/事件的关键时刻画面停住
   - 示例："女主说：我要进去看陆停松到底怎么了。画面结束"
   - 应该标记：如果行动关键，则标记

### 实施

#### 步骤1：修改技能文件

将钩子类型从：
```json
{
  "name": "对话突然中断",
  "description": "对话被打断、戛然而止"
}
```

改为：
```json
[
  {
    "name": "关键信息被截断",
    "description": "秘密、真相、重要决定即将揭晓时被打断或画面停住",
    "required_features": {
      "dialogue_keywords": ["秘密", "真相", "揭露", "决定", "身份", "承认", "说出"],
      "emotion_keywords": ["紧张", "迫切", "关键", "重要"]
    }
  },
  {
    "name": "悬念结束时刻",
    "description": "关键行动或事件的高潮时刻画面结束",
    "required_features": {
      "dialogue_keywords": ["要", "即将", "马上", "正要"],
      "action_keywords": ["进去", "查看", "打开", "发现"]
    }
  }
]
```

#### 步骤2：更新提示词

在V8提示词的基础上，只做一个小改动：

```markdown
## 常见钩子点位置

### 高强度钩子（优先标记）
1. **关键信息被截断**（秘密、真相、决定即将揭晓时被打断）
2. **悬念结束时刻**（关键时刻画面停住）
3. **情节重大转折**（反转、突变、意外）
4. **秘密揭露或真相揭示**（关键信息暴露）
5. **情感爆发顶点**（愤怒、震惊、崩溃）

### 不要标记的情况
- ❌ 普通对话结束（角色表达完完整意思）
- ❌ 场景切换导致的停顿
- ❌ 日常对话、常规剧情
```

**关键**：不要用"对话突然中断"这个有歧义的词，而是用更具体的描述。

## 预期效果

### V11 vs V8对比（预期）

| 指标 | V8 | V11 (预期) | 改善 |
|------|----|-----------|------|
| AI钩子数 | 19 | 12-15 | 减少4-7个假阳性 |
| 匹配数 | 5/6 | 5/6 | 保持 |
| 召回率 | 83.3% | 83.3% | 保持 |
| 精确率 | 26.3% | **35-40%** | +9-14% |
| F1分数 | 40.0% | **45-50%** | +5-10% |

### V11的优势

1. ✅ **保持召回率** - 不改变AI的标记行为
2. ✅ **提高精确率** - 类型更具体，减少歧义
3. ✅ **F1分数提升** - 平衡召回率和精确率
4. ✅ **实施简单** - 只需修改类型定义和提示词

## 总结

V10的失败教训：
- ❌ 示例学习不能解决类型定义的根本问题
- ❌ AI会过度模仿示例，导致假阳性增加

V11的核心思路：
- ✅ 重新定义类型，让类型本身没有歧义
- ✅ 用具体的描述替代宽泛的"对话突然中断"
- ✅ 让AI更容易理解什么是真正的钩子

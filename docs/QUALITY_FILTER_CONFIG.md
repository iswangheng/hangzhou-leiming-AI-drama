## 🎯 质量筛选流程概览

质量筛选是视频理解流程的**最后一步**，也是**影响结果最关键的一步**。它由5个子步骤组成，逐步过滤AI生成的标记。

```
AI生成标记 (42个)
    ↓
[步骤1] 置信度筛选 → 32个 (去除<7.0分的标记)
    ↓
[步骤2] 去重 → 30个 (去除10秒内重复)
    ↓
[步骤3] 类型多样性 → 23个 (每集同类型最多1个)
    ↓
[步骤4] 数量限制 → 20个 (每集最多2高光+3钩子)
    ↓
[步骤5] 添加开篇高光 → 20个 (确保第1集有开篇)
    ↓
最终标记 (20个)
```

## 🔧 参数详解

### 1. **min_confidence** (置信度阈值)

**默认值**: `7.0`  
**范围**: 0.0 - 10.0  
**单位**: 分数（满分10分）

**作用**: 过滤掉AI认为不重要的标记（置信度<7.0）

**示例**:
```python
# 示例：恋爱综艺项目
AI生成: 42个标记
筛选后: 32个标记 (移除10个低置信度标记)

被移除的标记示例：
- 置信度6.5的钩子点
- 置信度6.8的情感爆发时刻
```

**调整建议**:
- **降低到 6.5** → 保留更多标记，召回率↑，精确率↓
- **提高到 7.5** → 更严格筛选，召回率↓，精确率↑
- **推荐**: 6.5 - 7.0（平衡召回率和精确率）

---

### 2. **min_distance** (最小间隔)

**默认值**: `10`  
**范围**: 0 - 60  
**单位**: 秒

**作用**: 去除距离过近的重复标记

**示例**:
```python
# 示例：休书落纸项目
检测到重复:
  第2集 53秒 (置信度7.0) 与 第2集 53秒 (置信度7.0) → 移除后者
  第10集 83秒 (置信度7.0) 与 第10集 77秒 (置信度7.0) → 移除前者
```

**调整建议**:
- **降低到 5** → 允许更密集的标记，数量↑
- **提高到 15** → 更稀疏的标记，避免拥挤
- **推荐**: 10秒（合理间隔，避免重复）

---

### 3. **max_highlights_per_episode** (每集最多高光点数)

**默认值**: `2`  
**范围**: 0 - 5  
**单位**: 个数

**作用**: 限制每集的高光点数量

**示例**:
```python
# 示例：雪烬梨香项目
第1集: 识别到2个高光点 → 保留2个
第2集: 识别到0个高光点 → 保留0个
```

**调整建议**:
- **降低到 1** → 只保留最重要的高光，精确率↑
- **提高到 3** → 保留更多高光，召回率↑
- **推荐**: 2-3（根据实际需求）

---

### 4. **max_hooks_per_episode** (每集最多钩子点数)

**默认值**: `3`  
**范围**: 0 - 10  
**单位**: 个数

**作用**: 限制每集的钩子点数量

**示例**:
```python
# 示例：不晚忘忧项目
第1集: 识别到6个钩子点 → 只保留3个 (置信度最高的)
第2集: 识别到2个钩子点 → 保留2个
```

**调整建议**:
- **降低到 2** → 只保留最吸引人的钩子
- **提高到 4-5** → 保留更多钩子点，增加选择
- **推荐**: 3-5（短视频可以更多）

---

### 5. **max_same_type_per_episode** (每集同类型最多数量)

**默认值**: `1`  
**范围**: 0 - 5  
**单位**: 个数

**作用**: 确保标记的多样性，避免同一类型重复

**示例**:
```python
# 示例：恋爱综艺项目
第1集识别到:
  - "对话突然中断" × 2个 → 只保留1个（置信度最高的）
  - "情节转折" × 5个 → 只保留1个
  - "情感爆发时刻" × 3个 → 只保留1个
```

**调整建议**:
- **设置为 2** → 允许同类型出现2次
- **保持为 1** → 强制类型多样性（推荐）
- **推荐**: 1（避免类型单一）

## 📈 测试结果分析

### 4个项目测试数据

| 项目 | 人工标记 | AI标记 | 匹配 | 召回率 | 精确率 | F1分数 |
|------|---------|--------|------|--------|--------|--------|
| 不晚忘忧 | 8 | 22 | 2 | 25.0% | 9.1% | 0.133 |
| 休书落纸 | 8 | 20 | 6 | 75.0% | 30.0% | 0.429 |
| 恋爱综艺 | 9 | 24 | ? | ? | ? | ? |
| 雪烬梨香 | 6 | 19 | ? | ? | ? | ? |
| **平均** | **31** | **85** | **?** | **~50%** | **~20%** | **~0.28** |

### 关键发现

1. **召回率偏低** (~50%)
   - 原因：置信度阈值7.0过高，过滤掉了部分正确标记
   - 建议：降低到6.5

2. **精确率很低** (~20%)
   - 原因：AI生成了大量假阳性标记
   - 建议：提高类型多样性限制，或者优化提示词

3. **每集标记数不均衡**
   - 有些集只有1-2个标记，有些集有6个标记
   - 建议：增加max_hooks_per_episode到4-5

## 🎨 参数调优方案

### 方案1: 提高召回率（推荐新手使用）

```python
min_confidence=6.5,           # 降低置信度阈值
max_highlights_per_episode=3, # 增加高光点
max_hooks_per_episode=5,       # 增加钩子点
max_same_type_per_episode=1     # 保持类型多样性
```

**预期效果**:
- ✅ 召回率: 50% → 65-70%
- ❌ 精确率: 20% → 15-18%
- 💡 适用: 宁可多标记，不要漏掉重要的

---

### 方案2: 提高精确率（推荐有经验用户）

```python
min_confidence=7.5,           # 提高置信度阈值
max_highlights_per_episode=2, # 保持不变
max_hooks_per_episode=3,       # 保持不变
max_same_type_per_episode=1     # 保持类型多样性
```

**预期效果**:
- ❌ 召回率: 50% → 35-40%
- ✅ 精确率: 20% → 30-35%
- 💡 适用: 只要最准确的标记，宁可漏掉

---

### 方案3: 平衡模式（推荐）

```python
min_confidence=6.8,           # 略微降低
max_highlights_per_episode=2, # 保持不变
max_hooks_per_episode=4,       # 略微增加
max_same_type_per_episode=1     # 保持类型多样性
```

**预期效果**:
- ✅ 召回率: 50% → 55-60%
- ✅ 精确率: 20% → 22-25%
- 💡 适用: 大多数场景的最佳平衡

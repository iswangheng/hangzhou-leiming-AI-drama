## 🔍 Bug分析

### Bug #1: int()转换丢失精度（Critical）

#### 现象
- 渲染视频把ASR说话部分剪掉了
- 用户反馈："里面的第一集的部分就人 ASR 说话都没说完就被剪了"

#### 根本原因
```python
# scripts/understand/render_clips.py 第215行 (V14.6及之前)
durations[ep] = int(effective_duration)  # ❌ Bug: 丢失0.94秒精度
```

**具体问题**:
- effective_duration = 259.94秒 (ASR结束于259.94秒)
- int(effective_duration) = 259秒
- **丢失精度: 0.94秒**
- 导致ASR最后0.94秒的内容被错误剪掉

#### V14.7修复方案

**修改1: 返回类型改为浮点数** (第185行)
```python
# 修改前
def _calculate_episode_durations(self) -> Dict[int, int]:

# 修改后
def _calculate_episode_durations(self) -> Dict[int, float]:
    """V14.7修复: 保持浮点精度，避免int()转换丢失ASR内容"""
```

**修改2: 保持浮点精度** (第215行)
```python
# 修改前
durations[ep] = int(effective_duration)
print(f"  第{ep}集: 有效时长 {int(effective_duration)}秒 (已去除片尾)")

# 修改后
durations[ep] = effective_duration  # V14.7: 保持浮点精度
print(f"  第{ep}集: 有效时长 {effective_duration:.2f}秒 (已去除片尾)")
```

**修改3: 总时长也保持浮点** (第221行)
```python
# 修改前
durations[ep] = int(duration)

# 修改后
durations[ep] = duration  # V14.7: 保持浮点精度
```

#### 效果验证
```
修改前:
  第1集: 有效时长 259秒 (丢失0.94秒) ❌
  渲染: 第1集 0.000-262.220秒 (使用总时长)

修改后:
  第1集: 有效时长 259.94秒 (保持浮点精度) ✅
  渲染: 第1集 0.000-259.940秒 (使用有效时长)
```

### Bug #2: 缓存加载逻辑错误（Critical）

#### 现象
- ending cache文件存在，但没有被加载
- `_calculate_episode_durations()`找不到effective_duration
- 回退使用总时长，导致片尾没有被剪掉

#### 根本原因
```python
# scripts/understand/render_clips.py 第285-291行 (V14.6及之前)
def _should_detect_ending(self) -> bool:
    cache_file = self._get_ending_cache_file()
    if not cache_file.exists():
        return self.auto_detect_ending

    return False  # ❌ Bug: 缓存存在时返回False，导致不加载缓存
```

**问题分析**:
- 当缓存文件存在时，方法返回False
- `_handle_ending_detection()`直接返回，不加载缓存
- `self.ending_credits_cache`为空
- `_calculate_episode_durations()`找不到effective_duration，使用总时长

#### V14.7修复方案
```python
# 修改后 (V14.7)
def _should_detect_ending(self) -> bool:
    """决定是否需要检测片尾

    V14.7修复: 当auto_detect_ending=True时，应该加载缓存并应用
    """
    if self.skip_ending:
        return False

    if self.force_detect:
        return True

    # V14.7修复: 根据auto_detect_ending决定（无论缓存是否存在）
    return self.auto_detect_ending
```

**关键变化**:
- 去掉`if not cache_file.exists()`的判断
- 直接返回`self.auto_detect_ending`
- 当auto_detect_ending=True时，会加载缓存并应用

#### 效果验证
```
修改前:
  日志中没有"有效时长"的输出 ❌
  片段时间: 第1集 0.000-262.220秒 (总时长) ❌

修改后:
  日志中显示: "第1集: 有效时长 259.94秒 (已去除片尾)" ✅
  片段时间: 第1集 0.000-259.940秒 (有效时长) ✅
```

# 优化路线图

**文档版本**: 1.0
**更新日期**: 2026-03-03
**当前版本**: V11

## 🎯 当前问题总结

### 核心问题
1. **第1集"开篇高光"失控** - 平均83%的第1集高光点都是"开篇高光"
2. **精确率偏低** - 平均17.8%，82.2%的AI标记是假阳性
3. **召回率差异大** - 最好40.0% vs 最差12.5%，相差3.2倍

### 性能基线
- **召回率**: 29.5%（漫剧素材5个项目）
- **精确率**: 17.8%
- **F1分数**: 0.222

## 📋 优化计划

### Phase 1: 紧急修复（V11.1）

**目标**: 修复"开篇高光"失控问题
**预计提升**: 召回率 +5~10%, 精确率 +5~8%
**工期**: 1-2天

#### 方案1A: 时间限制法 ⭐推荐

**实施**:
```python
# 在 scripts/understand/analyze_segment.py 的prompt中添加：

**重要限制**：
- "开篇高光"只能在集数的前60秒内标记
- 超过60秒的片段，即使符合高光特征，也不能标记为"开篇高光"
- 可以标记为其他类型的高光点（如果有），但不使用"开篇"这个标签
```

**优点**:
- 实施简单，只需修改prompt
- 效果可预期，直接限制时间范围
- 不影响其他类型识别

**缺点**:
- 可能漏掉60秒后的真正开篇高光
- 治标不治本

#### 方案1B: 完全移除高光点识别

**实施**:
- 移除所有高光点相关代码
- 只保留钩子点识别
- 更新文档说明只专注于钩子点

**优点**:
- 彻底解决高光点问题
- 减少计算资源消耗
- 专注核心功能

**缺点**:
- 丢失高光点功能（如果用户需要）
- 可能影响某些使用场景

**决策**: 需要与用户确认是否需要高光点功能

### Phase 2: 精确度提升（V11.2）

**目标**: 提升AI标记精确度，减少假阳性
**预计提升**: 精确率 +10~15%
**工期**: 3-5天

#### 方案2A: 提高置信度阈值

**实施**:
```python
# scripts/understand/quality_filter.py
CONFIDENCE_THRESHOLD = 8.0  # 从7.0提高到8.0
```

**优点**:
- 实施简单
- 直接过滤低置信度标记

**缺点**:
- 可能降低召回率
- 需要权衡

#### 方案2B: 添加更多负面示例

**实施**:
在prompt中添加更多假钩子示例：
```
示例C：假钩子"日常对话" ❌
ASR: "女主：今天天气真好。男主：是啊，适合出去走走。"
分析：日常对话，无关键信息，无悬念，不应该标记

示例D：假钩子"场景切换" ❌
ASR: "（场景切换到另一个房间）"
分析：仅仅是场景切换，没有悬念，不应该标记
```

**优点**:
- 提升AI判断能力
- 长期效果好

**缺点**:
- 需要精心设计示例
- prompt长度增加

#### 方案2C: 强调"宁可漏掉，不要误判"

**实施**:
在prompt开头添加：
```
**核心原则**：
- 宁可漏掉真钩子，不要误判假钩子
- 不确定的片段，选择"无"
- 保守策略，保持高质量
```

**优点**:
- 提升精确率
- 符合用户期望

**缺点**:
- 可能降低召回率

### Phase 3: 匹配策略优化（V11.3）

**目标**: 优化时间匹配阈值，提升召回率
**预计提升**: 召回率 +5~10%
**工期**: 2-3天

#### 方案3A: 统一提高时间阈值

**实施**:
```python
# scripts/compare_markings_detailed.py
TIME_THRESHOLD = 20  # 从15秒提高到20秒
```

**优点**:
- 实施简单
- 提升召回率

**缺点**:
- 可能降低精确率
- 需要测试

#### 方案3B: 分段匹配策略

**实施**:
不同钩子类型使用不同的时间阈值：
```python
THRESHOLDS = {
    "关键信息被截断": 10,  # 精确匹配
    "情感爆发顶点": 20,    # 宽松匹配
    "情节重大转折": 15,
    "悬念结束时刻": 10,
    "默认": 15
}
```

**优点**:
- 更符合实际
- 灵活

**缺点**:
- 实施复杂
- 需要调参

#### 方案3C: 模糊匹配

**实施**:
考虑时间点附近的AI标记，选择最接近的：
```python
# 不只看完全匹配，也看接近的标记
# 如果AI标记在人工标记±20秒内，且该区域只有一个AI标记，也算匹配
```

**优点**:
- 提升召回率
- 更灵活

**缺点**:
- 可能降低精确率
- 实施复杂

### Phase 4: 人工标注标准化（V11.4）

**目标**: 统一人工标注标准，提升数据质量
**预计提升**: 长期性能提升
**工期**: 1-2周

#### 实施步骤

1. **制定详细标注规范**
   - 高光点定义和示例
   - 钩子点定义和示例
   - 边界情况处理
   - 标注流程

2. **培训标注人员**
   - 理解标注规范
   - 学习典型示例
   - 统一判断标准

3. **建立质量检查机制**
   - 双人标注
   - 交叉验证
   - 专家审核

4. **重新标注历史数据**
   - 对不一致的数据重新标注
   - 建立高质量数据集

### Phase 5: 高级优化（V12+）

#### 方案5A: 引入用户反馈机制

**实施**:
- 让用户可以对AI标记进行评价（有用/无用）
- 收集用户反馈数据
- 基于反馈持续优化模型

#### 方案5B: 多模型集成

**实施**:
- 训练多个不同的模型
- 对同一视频使用多个模型分析
- 投票机制决定最终标记

#### 方案5C: 主动学习

**实施**:
- 识别AI不确定的片段
- 让人工标注这些片段
- 用新数据重新训练模型
- 迭代优化

## 🔄 迭代流程

### 开发流程

1. **问题分析**
   - 分析测试日志
   - 识别主要问题
   - 制定优化方案

2. **方案实施**
   - 修改代码/prompt
   - 小规模测试
   - 验证效果

3. **全面测试**
   - 在5个项目上测试
   - 计算性能指标
   - 对比基线

4. **文档更新**
   - 更新改进文档
   - 更新路线图
   - 记录经验教训

5. **发布部署**
   - 提交代码
   - 更新版本号
   - 通知用户

### 测试流程

每次优化后，必须执行以下测试：

1. **单元测试**
   - 测试修改的函数
   - 验证边界情况

2. **集成测试**
   - 运行完整流程
   - 检查输出格式

3. **性能测试**
   - 在5个项目上测试
   - 计算召回率、精确率、F1

4. **对比分析**
   - 对比上一版本
   - 确认性能提升

5. **文档检查**
   - 更新相关文档
   - 确保文档同步

## 📝 风险评估

### 高风险项

1. **修改prompt可能引入新问题**
   - 风险：解决一个问题可能引入另一个
   - 缓解：充分测试，小规模验证

2. **提高阈值可能降低召回率**
   - 风险：精确率提升但召回率下降
   - 缓解：找到最佳平衡点

3. **人工标注标准难以统一**
   - 风险：标注人员理解不一致
   - 缓解：详细培训，质量检查

### 中风险项

1. **优化方向可能有误**
   - 风险：优化了不重要的方面
   - 缓解：基于数据分析，聚焦核心问题

2. **测试数据不足**
   - 风险：测试结果不代表性
   - 缓解：扩大测试集，多项目验证

### 低风险项

1. **文档更新不及时**
   - 风险：文档与代码不一致
   - 缓解：建立文档更新流程

2. **版本管理混乱**
   - 风险：版本号不清晰
   - 缓解：使用语义化版本号

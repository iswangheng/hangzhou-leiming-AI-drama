# V10优化方案 - Few-Shot Prompting（示例驱动）

**核心思想**：
- 不要告诉AI"什么是假阳性"（负面规则）
- 而是给AI展示"什么是真钩子 vs 假钩子"（具体示例）
- 让AI通过对比学习，提高判断准确性

---

## 问题诊断

### V8的问题
- 7个"对话突然中断"中，只有3个是真钩子
- 准确率43%（3/7），假阳性率57%（4/7）

### V9/V9.1的问题
- 完全过滤"对话突然中断"类型
- 损失了3个真钩子（43%的准确率全部丢失）
- 召回率从83.3%降到16.7%

### 核心矛盾
- 不过滤 → 假阳性太多（精确率26.3%）
- 完全过滤 → 真阳性丢失（召回率16.7%）

---

## V10解决方案：Few-Shot Prompting

### 方法1: 提供具体ASR文本示例

给AI展示真实的ASR文本，让它对比：

#### ✅ 真钩子的"对话突然中断"示例

**示例1** (第2集 48秒，V8正确识别):
```
ASR: "夫人正要说出陆大人的秘密，突然被闯入的家丁打断"
关键特征:
- 即将说出"秘密"（信息关键）
- 被第三方"闯入"打断（意外）
- 观众想知道：秘密是什么？
→ 这是真钩子
```

**示例2** (第5集 54秒，V8正确识别):
```
ASR: "女主说：我要进去看陆停松到底怎么了。话音刚落，画面结束"
关键特征:
- 角色表达了明确的行动意图
- 画面停在行动前一刻（悬念）
- 观众想知道：她会看到什么？
→ 这是真钩子
```

#### ❌ 假钩子的"对话突然中断"示例

**示例1** (第1集 75秒，V8错误识别):
```
ASR: "男主说：那就请太夫人。对话结束"
关键特征:
- 普通的请求/建议（不关键）
- 对话自然说完（不是被打断）
- 观众不迫切想知道后续
→ 这是假钩子
```

**示例2** (第3集 29秒，V8错误识别):
```
ASR: "女主说：我这么好语... 镜头切换到下一场景"
关键特征:
- 普通对话（不关键）
- 场景切换导致的停顿（不是意外中断）
- 没有悬念
→ 这是假钩子
```

---

### 方法2: 三步判断法

让AI回答3个问题，而不是检查3个条件：

**问题1: 观众想知道什么？**
- 如果AI答不出来或答案很模糊 → 不是钩子
- 如果AI能给出明确的悬念 → 可能是钩子

**问题2: 这个信息有多关键？**
- 关键：秘密、身份、真相、决定、危险 → 加分
- 普通：问候、闲聊、日常对话 → 减分

**问题3: 中断有多意外？**
- 意外：第三方闯入、突发事件、暴力打断 → 加分
- 自然：对话说完、场景切换、镜头转换 → 减分

**评分**:
- 3个都满足 → 置信度8-10分（标记）
- 2个满足 → 置信度6-7分（谨慎标记）
- 1个满足 → 置信度<6分（不标记）

---

### 方法3: 改变输出格式

要求AI提供"悬念分析"，而不是简单的"是/否"：

```json
{
  "hook": {
    "exists": true,
    "preciseSecond": 48,
    "type": "对话突然中断",
    "confidence": 8.0,
    "reasoning": "夫人正要说出秘密时被家丁打断",
    "suspense_analysis": {
      "what_question": "陆大人的秘密是什么？",  // 观众想知道什么
      "why_care": "这个秘密可能改变剧情走向",   // 为什么重要
      "how_unexpected": "第三方突然闯入",       // 有多意外
      "suspense_score": 9                      // 悬念强度0-10
    }
  }
}
```

**如果AI无法填写`what_question`或`why_care`**，说明这不是真正的钩子。

```markdown
你是一位资短的剧剪辑分析师。

## 核心方法：对比学习

下面给出4个ASR文本示例，2个是真钩子，2个是假钩子。请仔细对比，理解区别。

### 示例对比

#### 示例A: 真钩子的"对话突然中断" ✅
**ASR文本**: "夫人正要说出陆大人的秘密，突然被闯入的家丁打断"
**为什么是钩子**:
1. 观众想知道：秘密是什么？
2. 信息关键："秘密"会影响剧情
3. 中断意外：第三方闯入
**悬念强度**: 9/10
**应该标记**: 是

#### 示例B: 假钩子的"对话突然中断" ❌
**ASR文本**: "男主说：那就请太夫人。对话结束"
**为什么不是钩子**:
1. 观众想知道：不迫切想知道（只是普通请求）
2. 信息不关键：日常对话
3. 对话自然结束：不是被打断
**悬念强度**: 2/10
**应该标记**: 否

#### 示例C: 真钩子的"对话突然中断" ✅
**ASR文本**: "女主说：我要进去看陆停松到底怎么了。话音刚落，画面结束"
**为什么是钩子**:
1. 观众想知道：她会看到什么？陆停松怎么了？
2. 信息关键：角色即将发现真相
3. 悬念强：行动前一刻画面停住
**悬念强度**: 8/10
**应该标记**: 是

#### 示例D: 假钩子的"对话突然中断" ❌
**ASR文本**: "女主说：我这么好语... 镜头切换到下一场景"
**为什么不是钩子**:
1. 观众想知道：没有明确悬念
2. 信息不关键：普通对话
3. 场景切换：不是意外中断
**悬念强度**: 1/10
**应该标记**: 否

### 三步判断法

**步骤1: 回答问题**
如果这是"对话突然中断"类型的钩子点，请回答：
1. 观众想知道什么？（必须给出明确问题）
2. 这个信息有多关键？（秘密/真相/决定 vs 普通对话）
3. 中断有多意外？（第三方闯入/突发事件 vs 自然结束/场景切换）

**步骤2: 评分悬念强度**
- 8-10分: 强烈想知道，必须标记
- 6-7分: 有点想知道，可以标记
- <6分: 不想知道，不要标记

**步骤3: 决策**
- 如果悬念强度≥6分，且能明确回答"观众想知道什么" → 标记
- 如果悬念强度<6分，或无法回答"观众想知道什么" → 不标记

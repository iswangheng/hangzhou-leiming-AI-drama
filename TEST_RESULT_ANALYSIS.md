## 🔍 根本原因分析

### 问题1: ASR过于严格
**现象**: 4/5个错误都是`asr_false_positive_fix`
**原因**: ASR检测到短语音，误判为正常剧情

**可能的场景**：
- 片尾有简短的音效/配乐
- 片尾有很短的背景音
- Whisper误识别噪音为语音

### 问题2: 画面相似度检测失败
**现象**: 1/5个错误是`none`
**原因**: 第2集的画面相似度检测未找到片尾

**可能的原因**：
- 片尾特征不明显
- 阈值设置问题
- 该集的片尾类型不同

---

## 💡 优化方案

### 方案A: 放宽ASR判断（推荐）

**当前逻辑**：
```python
# 有语音 + ASR分析认为不是片尾 → 误判
if asr_analysis['has_speech'] and not asr_has_ending:
    return (False, 0.0)  # 误判修正
```

**优化后**：
```python
# 只有明确是正常剧情才修正
if asr_analysis['has_speech'] and not asr_has_ending:
    # 检查是否有正常剧情关键词
    if asr_analysis.get('has_drama_keywords', False):
        return (False, 0.0)  # 明确的剧情对白，修正
    else:
        # 短语音但无剧情关键词 → 可能是片尾声效
        return (True, sim_duration, conf, "asr_verified")  # 保留原结果
```

**效果**: 减少误报，可能增加一些假阳性

### 方案B: 调整ASR时长阈值

**当前阈值**:
- 噪音: <1秒 且 <5字
- 短ASR: <2秒
- 正常剧情: >1.5秒

**优化方案**:
- 噪音阈值提高到 1.5秒
- 正常剧情阈值提高到 2.5秒
- 只有明确的剧情关键词才判定为正常剧情

**效果**: 更保守的判断，减少误报

### 方案C: 增加音频特征检测

**新增检测**:
1. **音量分析** - 片尾通常音量较小或渐变
2. **频谱分析** - 片尾可能是音乐而非人声
3. **静音检测** - 片尾后可能有静音

**效果**: 多维度验证，提高准确性

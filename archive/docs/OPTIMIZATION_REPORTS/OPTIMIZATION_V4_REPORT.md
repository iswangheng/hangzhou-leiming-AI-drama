## 📊 执行摘要

### V4.0 优化内容

本次优化主要针对**分析窗口大小**进行调整，这是基于V3测试结果发现的核心问题。

**核心修改**：
- 分析窗口：60秒 → **30秒** (缩小50%)
- 重叠时间：10秒 → **5秒** (减少50%)
- 目的：提高AI识别精度，减少窗口过大导致的信息模糊

### 测试结果对比

| 指标 | V3 (60秒窗口) | V4 (30秒窗口) | 变化 |
|------|--------------|--------------|------|
| 总人工标记 | 44 | 44 | - |
| 总AI标记 | ~10 | 14 | +40% |
| 总匹配数 | ~11 | 13 | +18% |
| **召回率** | **~25%** | **29.5%** | **+4.5%** |
| **精确率** | **100%** | **92.9%** | **-7.1%** |

## 🚀 V4.0 修改详情

### 修改的文件

**唯一修改**: `scripts/understand/extract_segments.py`

```python
# V3配置 (60秒窗口)
SEGMENT_DURATION = 60  # 片段时长（秒）
SEGMENT_OVERLAP = 10   # 重叠时长（秒）

# V4配置 (30秒窗口)
SEGMENT_DURATION = 30  # 片段时长（秒）- V4优化：缩小窗口提高精度
SEGMENT_OVERLAP = 5    # 重叠时长（秒）- V4优化：减少重叠
```

### 优化原理

**问题诊断**：
1. V3的60秒窗口太大，包含太多情节转折
2. AI难以在60秒内精确定位关键时刻
3. 多个钩子点可能都在同一个60秒窗口中，导致AI只识别一个

**V4改进**：
1. 30秒窗口更聚焦，AI更容易识别关键时刻
2. 片段数量从23个增加到41个（百里将就项目）
3. 提高了时间分辨率和识别精度

## 🔍 关键发现

### 1. 召回率仍然偏低 ⭐⭐⭐⭐⭐

**现象**：
- 总召回率只有29.5%，意味着AI遗漏了70.5%的人工标记
- AI只输出了14个标记，而人工有44个

**可能原因**：
1. **AI过于保守**：Prompt中"宁缺毋滥"原则让AI不敢标记
2. **类型匹配困难**：AI无法将识别的钩子映射到31种类型
3. **特征提取不足**：5张关键帧可能不够表达情节转折
4. **置信度阈值过高**：7.0的阈值可能过滤掉了很多正确标记

**证据**：
- 所有项目的精确率都很高（75%-100%），说明AI标记的都很准确
- 但召回率很低（25%-37.5%），说明AI标记得太少

### 2. 类型识别问题 ⭐⭐⭐⭐

**现象**：
- 部分AI标记的`hook_type`为`null`
- AI说明"无法匹配到技能文件中的类型"

**示例**（百里将就）：
```json
{
  "timestamp": 77,
  "episode": 2,
  "type": null,  // ← 无法识别类型
  "confidence": 9.0
}
```

**分析**：
- 31种钩子类型可能太多，AI难以准确映射
- 或者类型定义不够清晰，边界模糊

### 3. 数据预处理问题 ⭐⭐⭐

**现象**：
```
警告: 第2集时长为0，跳过
警告: 第4集时长为0，跳过
警告: 第5集时长为0，跳过
```

**影响**：
- 多个项目的很多集数据没有加载成功
- 导致可分析的片段数量减少
- 进一步降低了召回率

**根本原因**：
- 关键帧提取或ASR转录可能失败
- 视频文件可能损坏
- 预处理脚本可能有问题

### 4. 时间精度良好 ⭐⭐

**现象**：
- 匹配对的时间偏差大多在10秒以内
- 没有超过30秒容差的匹配

**结论**：
- 30秒窗口已经提供了足够的时间精度
- AI识别的时间戳基本准确

## 💡 优化建议

### P0 - 紧急优化（影响最大）

#### 1. 调整Prompt策略 ⭐⭐⭐⭐⭐

**问题**：AI过于保守，标记数量太少

**建议**：
- 移除或弱化"宁缺毋滥"原则
- 改为"宁可多标记，不要遗漏"
- 降低标记的门槛，让AI更倾向于标记

**预期效果**：召回率从29.5%提升到50%+

#### 2. 简化类型系统 ⭐⭐⭐⭐⭐

**问题**：31种类型太多，AI无法准确匹配

**建议**：
- 将31种钩子类型合并为10-15种核心类型
- 或者完全移除类型要求，只识别高光/钩子，不要求细分类型
- 保留类型作为元数据，但不作为必需字段

**预期效果**：减少null类型，提高AI标记积极性

#### 3. 修复数据预处理 ⭐⭐⭐⭐

**问题**：多集数据时长为0，无法分析

**建议**：
- 检查关键帧提取脚本
- 检查ASR转录脚本
- 确保所有视频都能成功预处理

**预期效果**：增加可分析的数据量，提高召回率

### P1 - 短期优化（1-2周）

#### 4. 降低置信度阈值 ⭐⭐⭐

**当前**：7.0
**建议**：6.0 或 5.5

**理由**：
- 当前精确率92.9%很高，说明可以适当降低阈值
- 可能会引入一些低质量标记，但能提高召回率

#### 5. 增加关键帧数量 ⭐⭐⭐

**当前**：每片段5张关键帧
**建议**：增加到10张

**理由**：
- 30秒窗口内，10张关键帧能提供更丰富的视觉信息
- 帮助AI更好地理解情节转折

### P2 - 长期优化（1个月+）

#### 6. 进一步缩小窗口 ⭐⭐

**考虑**：20秒或15秒窗口

**风险**：
- 可能会错过一些情节转折
- 计算量增加

#### 7. 添加上下文分析 ⭐⭐

**思路**：
- 分析前后片段的内容
- 理解情节连贯性
- 更准确地识别钩子点

- 优化：30秒分析窗口

- 发现：AI过于保守是主要瓶颈

## ✅ 结论

### V4.0 达成的目标

1. ✅ **缩小分析窗口**：从60秒改为30秒
2. ✅ **提高时间精度**：时间偏差在±30秒内
3. ✅ **小幅提升召回率**：从25%提升到29.5%
4. ✅ **保持高精确率**：92.9%

### 核心问题仍然存在

**召回率偏低（29.5%）**的根本原因：
1. ❌ AI过于保守（Prompt问题）
2. ❌ 类型系统过于复杂（31种类型）
3. ❌ 数据预处理不完整（多集数据缺失）

### 下一步优先级

**最高优先级（P0）**：
1. 调整Prompt，让AI更积极地标记
2. 简化或移除类型要求
3. 修复数据预处理问题

**预期效果**：
- 召回率从29.5%提升到50%+
- AI标记数量从14个增加到30-40个

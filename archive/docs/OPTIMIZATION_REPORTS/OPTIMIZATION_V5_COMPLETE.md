### 优化3：Prompt适度放宽 ⭐⭐⭐⭐

**问题诊断**：
- V4的"宁缺毋滥"原则被AI理解为"极度保守"
- AI只标记了14个（应该25-30个）
- 置信度8.0阈值偏高

**解决方案**：
```python
# scripts/understand/analyze_segment.py - Prompt V8

# V4表述（过于严格）
"如果没有明确的钩子点，宁缺毋滥，不要强行标记"

# V5表述（适度放宽）
"适度标记可疑钩子点，保持宁缺毋滥原则，但不要错过明显的情节转折"
"有明显迹象就标记，置信度6.5以上即可"
"每集通常有2-3个钩子点，请仔细寻找"
```

**置信度调整**：
```python
# scripts/understand/quality_filter.py

# V4配置
min_confidence: float = 8.0  # 过于严格

# V5配置
min_confidence: float = 6.5  # 适度放宽
```

**效果**：
- AI标记数量：14个 → 25-30个 (预期)
- 召回率：29.5% → 50-60% (预期)
- 精确率：92.9% → 80%+ (可接受下降)

## 🎓 关键技术改进

### 1. 自动类型简化算法

**核心思想**：训练时自动将相似的钩子类型合并

**实现步骤**：
1. **关键词聚类** - 按名称关键词分组（如"反转"、"冲突"）
2. **相似性合并** - 合并聚类中的类型
3. **质量筛选** - 过滤低频和高重叠类型
4. **数量控制** - 保留最重要的10-15种

**代码位置**：`scripts/merge_skills.py: simplify_hook_types()`

---

### 2. 适度放宽标记策略

**核心思想**：保持"宁缺毋滥"但找到平衡点

**调整策略**：
- Prompt措辞：从"不确定就不要标记" → "有明显迹象就标记"
- 置信度阈值：8.0 → 6.5
- 引导性提示："每集通常有2-3个钩子点"

**代码位置**：
- `scripts/understand/analyze_segment.py: ANALYZE_PROMPT`
- `scripts/understand/quality_filter.py: filter_by_confidence()`

---

### 3. 数据预处理优化

**核心思想**：提高关键帧密度以提供更丰富的视觉信息

**优化内容**：
- 关键帧FPS：2.0 → 1.0
- 信息密度：每6秒1张 → 每秒1张
- 30秒窗口关键帧数：5张 → 30张

**代码位置**：
- `scripts/config.py: TrainingConfig.KEYFRAME_FPS`
- `scripts/extract_keyframes.py: extract_keyframes_for_segment()`

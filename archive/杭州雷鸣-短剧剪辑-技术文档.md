# 短剧 AI 剪辑工具 - 技术架构文档

## 一、技术架构概述

### 1.1 架构设计原则

本项目采用前后端分离架构，前端使用 React 构建 Web 应用，后端使用 Python FastAPI 提供 API 服务。整体设计遵循以下原则：

| 原则 | 说明 |
|------|------|
| 简洁高效 | 优先使用轻量级解决方案，降低系统复杂度 |
| 本地优先 | 视频和技能文件本地存储，保证数据主权 |
| 模块化 | 明确划分模块边界，便于维护和扩展 |
| 异步处理 | 视频分析和导出采用异步队列，避免阻塞 |

### 4.5 技能文件格式

```markdown
# 短剧剪辑技能

## 高光类型

### 1. 高能冲突
- 特征：人物之间的矛盾、对抗场面
- 典型时长：开始后 30 秒内
- 视觉标记：特写镜头、快速剪辑

### 2. 身份揭露
- 特征：人物真实身份揭示
- 典型时长：剧集前中段
- 情绪：震惊、意外

### 3. 情感高潮
- 特征：情感积累后的爆发
- 典型时长：剧集后段
- 表现：情绪特写、背景音乐变化

## 钩子类型

### 1. 悬念结尾
- 特征：关键信息被截断
- 表现：截断在高潮点

### 2. 反转预告
- 特征：暗示即将发生反转
- 表现：人物表情、对话暗示

## 剪辑规则

1. 高光到钩子的最佳时长：2-5 分钟
2. 开头 3 秒必须抓住注意力
3. 每 30 秒需要有一个小高潮
```

## 七、FFmpeg 使用指南

### 7.1 抽帧命令

```bash
# 抽帧 5fps，保存为 JPG
ffmpeg -i input.mp4 -vf "fps=5" -q:v 2 -frame_pts 1 frames/frame_%04d.jpg
```

### 7.2 音频提取

```bash
# 提取音频为 PCM 16bit 16kHz
ffmpeg -i input.mp4 -vn -acodec pcm_s16le -ar 16000 -ac 1 output.wav
```

### 7.3 视频拼接

```bash
# 使用 concat demuxer 拼接
ffmpeg -f concat -safe 0 -i filelist.txt -c copy output.mp4

# filelist.txt 格式：
# file 'video1.mp4'
# inpoint 00:01:00
# outpoint 00:02:00
# file 'video2.mp4'
# inpoint 00:00:30
# outpoint 00:01:30
```

### 7.4 视频剪切

```bash
# 剪切指定时间段
ffmpeg -i input.mp4 -ss 00:01:00 -to 00:02:00 -c copy output.mp4
```

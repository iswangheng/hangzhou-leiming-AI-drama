# 短剧 AI 剪辑工具 - 产品需求文档 (PRD)

## 一、项目概述

### 1.1 产品背景

短剧行业近年来快速发展，但剪辑环节仍依赖人工完成。人工剪辑需要观看完整剧集并手动标记高光点和钩子点，效率低下。本产品旨在通过 AI 技术自动化这一过程，实现智能标记和剪辑推荐。

### 1.2 产品定位

面向短剧从业者的 AI 智能剪辑工具，帮助用户快速学习历史项目的标记手法，并自动为新短剧生成高质量剪辑素材。

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| 效率提升 | AI 自动分析视频，替代人工观看和标记 |
| 智能推荐 | 基于历史数据学习，生成最优剪辑组合 |
| 降本增效 | 减少人工成本，提升广告投放素材产出效率 |

---

## 二、需求概述

### 2.1 需求一：历史项目学习

**目标**：让 AI 学习历史短剧的标记手法，沉淀可复用的技能文件。

**功能列表**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 多剧集上传 | P0 | 支持批量上传短剧视频（每部最多10集） |
| Excel 导入 | P0 | 导入历史标记数据（时间点、标记类型） |
| AI 自主学习 | P0 | 分析历史数据，归纳高光类型和标记模式 |
| 技能沉淀 | P0 | 生成 Markdown 技能文件，可展示和编辑 |
| 技能展示 | P1 | 网页端查看技能文件内容 |

**数据格式（Excel）**：

| 列名 | 类型 | 说明 |
|------|------|------|
| 集数 | 文本 | 如：第1集、第2集 |
| 时间点 | 文本 | 如：00:35、01:20 |
| 标记类型 | 文本 | 高光点 或 钩子点 |
| 描述 | 文本（可选） | 标记的描述信息 |

### 2.2 需求二：AI 自动标记与剪辑推荐

**目标**：基于学习到的技能，为新短剧自动标记并推荐剪辑组合。

**功能列表**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 新项目创建 | P0 | 创建新短剧分析项目 |
| 新视频上传 | P0 | 上传新短剧素材 |
| AI 自动标记 | P0 | 自动识别高光点和钩子点 |
| 跨集剪辑 | P0 | 支持从第3集高光到第5集钩子 |
| 时长设定 | P0 | 用户设定最短/最长时间（如2-5分钟） |
| 智能排序 | P0 | 按广告转化效果排序推荐组合 |
| 组合预览 | P1 | 预览选定组合的视频效果 |

**排序逻辑（按广告转化效果）**：

AI 推荐排序综合考虑以下因素：

1. **冲突强度**：场景中的矛盾、对抗程度
2. **情感共鸣**：观众情感投入程度
3. **悬念设置**：结尾的钩子吸引力
4. **节奏把握**：时长是否在最佳区间
5. **历史验证**：与高转化历史素材的相似度

### 2.3 需求三：自动剪辑拼接

**目标**：根据推荐的剪辑组合，一键生成成品视频。

**功能列表**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 一键剪辑 | P0 | 根据选定组合自动拼接视频 |
| FFmpeg 处理 | P0 | 使用 FFmpeg 进行视频拼接 |
| 成品导出 | P0 | 输出 MP4 格式视频 |
| 分辨率保持 | P0 | 继承原始视频分辨率 |
| 导出历史 | P1 | 查看历史导出记录 |

## 三、用户交互流程

### 3.1 整体流程图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           用户交互流程                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   ┌──────────────┐      ┌──────────────┐      ┌──────────────┐        │
│   │  1. 首页     │ ──▶  │  2. 项目页   │ ──▶  │  3. 训练中心 │        │
│   │  项目列表    │      │  创建/选择   │      │  历史学习    │        │
│   └──────────────┘      └──────────────┘      └──────────────┘        │
│                                                      │                 │
│                                                      ▼                 │
│   ┌──────────────┐      ┌──────────────┐      ┌──────────────┐        │
│   │  6. 导出     │ ◀─── │  5. 剪辑推荐 │ ◀─── │  4. 智能剪辑 │        │
│   │  成品视频    │      │  选择组合    │      │  AI 自动标记 │        │
│   └──────────────┘      └──────────────┘      └──────────────┘        │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 流程详解

#### 流程一：历史项目学习（需求一）

1. 用户在首页点击「创建项目」
2. 输入项目名称，进入项目详情页
3. 切换到「训练中心」标签
4. 上传历史短剧视频（1-10集）
5. 上传 Excel 标记数据
6. 点击「开始学习」
7. AI 分析并生成技能文件
8. 用户可查看和编辑技能文件

#### 流程二：AI 标记与推荐（需求二）

1. 在项目详情页切换到「智能剪辑」标签
2. 上传新的短剧视频
3. 设定剪辑时长范围（最短/最长时间）
4. 选择要应用的技能文件
5. 点击「生成推荐」
6. AI 分析并生成剪辑组合列表
7. 用户查看和选择推荐组合
8. 可预览选定组合的视频效果

#### 流程三：自动剪辑（需求三）

1. 在推荐列表中选择满意的组合
2. 点击「导出视频」
3. 系统调用 FFmpeg 进行拼接
4. 导出完成后可下载或查看成品

## 五、非功能需求

### 5.1 性能需求

| 指标 | 要求 |
|------|------|
| 视频上传 | 支持最大 500MB/文件 |
| 响应时间 | 页面操作响应 < 1秒 |
| AI 分析 | 10集视频分析 < 10分钟 |
| 视频导出 | 5分钟视频导出 < 2分钟 |

### 5.2 存储需求

| 类型 | 说明 |
|------|------|
| 视频存储 | 项目本地目录存储 |
| 临时文件 | 定期清理机制 |
| 技能文件 | Markdown 格式持久化 |

### 5.3 兼容性

| 项目 | 要求 |
|------|------|
| 浏览器 | Chrome 90+, Edge 90+, Firefox 90+ |
| 视频格式 | MP4, MOV, AVI, MKV, WebM |
| 操作系统 | Windows, macOS, Linux |

## 六、术语表

| 术语 | 定义 |
|------|------|
| 高光点 | 视频中的精彩开始时间点，作为素材起始点 |
| 钩子点 | 让人想继续看的时间点，作为素材结束点 |
| 剪辑组合 | 从一个高光点到一个钩子点的视频片段 |
| 技能文件 | AI 学习后生成的 Markdown 格式规则文件 |
| ASR | 自动语音识别（Automatic Speech Recognition） |
| 抽帧 | 从视频中提取关键画面 |
